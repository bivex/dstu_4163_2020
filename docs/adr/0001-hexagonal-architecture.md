# ADR-0001: Гексагональна архітектура з доменом, керованим Catala-специфікацією

Статус: прийнято · Дата: 2026-06-13

## Контекст

Маємо формальну, виконувану специфікацію ДСТУ 4163:2020 у Catala (13 scope).
Потрібна Python-реалізація («Dilovod4»), яку можна повторно використати в CLI,
веб-API та інтеграції з СЕД без переписування бізнес-логіки.

## Рішення

1. **Гексагональна (ports & adapters) архітектура**, чотири шари:
   `domain` → `application` → (`infrastructure`, `presentation`).
   Залежності спрямовані всередину; зовнішні шари залежать від внутрішніх.

2. **Домен — джерело істини, дзеркало Catala**. Кожен Catala-scope стає одним
   доменним правилом (`ConformanceRule`). Одне правило — одна причина зміни (SRP).
   Правила чисті: документ на вхід, `RuleResult` на вихід, без IO.

3. **Розмежування понять**:
   - інваріанти (структурна валідність) → виняток у конструкторі агрегату;
   - відповідність нормі → `Finding` у звіті, не виняток.

4. **Порти в домені, адаптери ззовні**: `RuleSetProvider`,
   `DocumentRepository` оголошені як `Protocol`/ABC у домені; реалізації — в
   `infrastructure`. Домен не знає про БД/мережу/файли.

5. **Use-case шар** (`ValidateDocument`) оркеструє: завантажити правила з порту,
   застосувати, зібрати звіт. Залежності — через конструктор (DIP, без service
   locator, без глобальних синглтонів).

6. **DTO на межах**: вхід/вихід застосунку — прості dataclass-DTO; доменні
   сутності не протікають у presentation/infrastructure.

## Наслідки

- (+) Додати правило = додати клас, що реалізує порт правила (OCP); ядро стабільне.
- (+) Домен тестується без інфраструктури; адаптери — окремими інтеграційними тестами.
- (+) Заміна сховища/інтерфейсу не зачіпає домен.
- (−) Більше файлів і явних контрактів, ніж у «скрипті». Свідомий компроміс заради
  розширюваності та тестопридатності.

## Структура каталогів

```
src/dilovod4/
  domain/            # моделі, value objects, агрегат, порти, правила (чистий Python)
    model/           # enums, value objects, Document
    rules/           # 13 правил відповідності + базовий протокол
    ports.py         # інтерфейси (RuleSetProvider, DocumentRepository)
    errors.py        # DomainError та підтипи
    events.py        # доменні події
  application/       # use-cases, DTO, порти застосунку
  infrastructure/    # адаптери: config, репозиторій, провайдер правил
  presentation/      # CLI
```
