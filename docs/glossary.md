# Глосарій / Ubiquitous Language — Dilovod4

Спільна мова між нормою (ДСТУ 4163:2020) і кодом. Кожен термін має українську
назву з норми та ASCII-ідентифікатор у коді.

## Сутності предметної області

| Українською (норма) | Код (identifier) | Визначення |
|---|---|---|
| Документ | `Document` | Корінь агрегату. Носій реквізитів та параметрів оформлення, що підлягають перевірці на відповідність ДСТУ 4163:2020. |
| Реквізит | `Requisite` / `RequisiteSet` | Обовʼязковий або дозволений елемент оформлення (найменування, дата, підпис тощо). Кодуються 1..32 за нормою. |
| Бланк | `BlankType` | Аркуш із нанесеними постійними реквізитами: загальний, листа, конкретного виду. |
| Поля сторінки | `PageMargins` | Ліве/праве/верхнє/нижнє поле в мм. |
| Формат паперу | `PaperFormat` | A4 / A5 / A3. |
| Типографіка | `Typography` | Гарнітура, кеглі (pt), довжина рядка реквізитів. |
| Відступи від лівого поля | `LeftIndents` | Абзац / адресат / гриф / розшифрування підпису (мм). |
| Строк зберігання | `StorageTerm` | Тимчасовий / тривалий / постійний — визначає бік друку. |
| Бік друку | `PrintSide` | Один бік / обидва боки аркуша. |
| Спосіб дати | `DateStyle` | Цифровий / зворотний цифровий / словесно-цифровий. |
| Розміри символів | `SymbolDimensions` | Герб, емблема, QR-код, зона держреєстрації (мм). |

## Поняття перевірки (наша надбудова над нормою)

| Термін | Код | Визначення |
|---|---|---|
| Правило відповідності | `ConformanceRule` | Специфікація одного параграфа ДСТУ. Одне правило — одна причина зміни (SRP). |
| Знахідка | `Finding` | Конкретне порушення/зауваження, що його повертає правило. |
| Результат правила | `RuleResult` | Підсумок застосування одного правила до документа. |
| Звіт відповідності | `ConformanceReport` | Сукупність результатів усіх правил; `conforms` = всі правила пройдені. |
| Інваріант | invariant | Структурне обмеження, без якого документ не може існувати (нп. кількість ≥ 0). Перевіряється у конструкторі агрегату, окремо від відповідності нормі. |

## Розмежування понять

- **Структурна валідність** (інваріанти) — чи може документ взагалі існувати.
  Порушення → виняток у домені (`DomainError`).
- **Відповідність нормі** (правила) — чи відповідає документ ДСТУ 4163:2020.
  Порушення → `Finding` у звіті, не виняток. Це бізнес-результат, а не помилка.

## Відповідність Catala-специфікації

Кожне доменне правило 1:1 відображає `scope` із `dstu_4163_2020.catala_en`.
Carta відповідності — у `docs/requirements.md`.
